## 📁 รูปแบบไฟล์

แต่ละไฟล์เป็น CSV บีบอัด gzip (`.csv.gz`) ตั้งชื่อตามรูปแบบ `{symbol}_1m_{year}.csv.gz`

มี columns:

- `timestamp` — Unix timestamp
- `datetime` — เวลา (UTC)
- `open`, `high`, `low`, `close` — ราคา (THB)
- `volume` — ปริมาณการเทรด

## 🐍 วิธีโหลดข้อมูล (Python)

```python
import pandas as pd
import glob

# อ่านไฟล์เดียว (gzip อ่านได้เลย ไม่ต้องแตกไฟล์)
df = pd.read_csv('btc_thb_1m_2024.csv.gz')

# รวมทุกปีของ BTC
files = sorted(glob.glob('btc_thb_1m_*.csv.gz'))
btc = pd.concat([pd.read_csv(f) for f in files], ignore_index=True)

# รวมทุกปีของ ETH
files = sorted(glob.glob('eth_thb_1m_*.csv.gz'))
eth = pd.concat([pd.read_csv(f) for f in files], ignore_index=True)
```

## 📈 คุณภาพข้อมูล

ข้อมูลผ่านการตรวจสอบแล้ว: ไม่มี duplicate, ไม่มีราคาผิดปกติ (≤ 0), OHLC ถูกต้องตามหลัก (high ≥ low ฯลฯ)

ความครบถ้วนของข้อมูลขึ้นอยู่กับปริมาณการเทรดในแต่ละช่วง:

- **2021 เป็นต้นไป** — ข้อมูลค่อนข้างครบ (82-98%) เหมาะกับ backtest
- **2018-2020** — volume ต่ำ มี gap มาก (20-66%) ควรใช้อย่างระมัดระวัง

Gap ที่เกิดขึ้นมาจากนาทีที่ไม่มีการเทรด ซึ่ง Bitkub จะไม่บันทึกแท่งนั้น ส่วนใหญ่เป็น gap สั้น (2-5 นาที)
